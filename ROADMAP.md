# Icon Creator Roadmap

## 背景说明

上一轮已完成的能力主要包括：

- 顶部“插入”分类弹窗与左侧拖拽上图
- 画布对象 `ContextMenu` 基础能力
- `Iconify` 默认浏览与懒加载优化
- 色板 / 渐变预设管理弹窗
- `Canvas / SVG` 基础模式切换
- 空格拖动画布与 `Ctrl + 鼠标滚轮` 缩放

在此基础上，本轮规划进入“编辑工作台第二阶段”，重点解决以下问题：

- 工作区布局仍不够可伸缩，左侧面板占用空间固定
- `SVG` 入口能力偏弱，尚未形成“图形视图 / 代码视图”的完整体验
- 画布对象右键与图层右键行为还不完全统一
- 右侧“检查 / 预览”占据固定面板，不够轻量
- 缺少多项目并行编辑能力，无法像专业设计软件一样多标签切换
- 画布视口交互仍存在滚动与拖动限制
- 预设管理器仍有冗余说明，布局密度和配置能力需要继续优化

---

## 本轮目标概述

本轮规划聚焦 7 个方向：

1. **左侧面板可隐藏化**
   - 左侧 `panel` 支持收起 / 展开
   - 入口放在编辑工具栏底部
   - 默认展开，并补齐过渡动画

2. **Canvas / SVG 视图升级**
   - `SVG` 按钮补充 `tooltips`
   - 支持切换为“代码模式”和“图形模式”
   - 代码模式只读预览 SVG 源码并提供代码高亮

3. **右键菜单行为统一**
   - 画布图形右键时，复用图层右键菜单能力
   - 保持对象操作入口一致

4. **检查 / 预览入口轻量化**
   - 从右侧固定面板中移出
   - 以图标形式放到 `Canvas / SVG` 切换器左侧
   - 通过 `tooltips` 展示问题列表，无问题时提供空状态

5. **多标签项目工作流**
   - 画布顶部升级为标签栏
   - 每个标签代表一个独立项目
   - 交互体验参考 Photoshop 的标签页模式

6. **画布视口交互修复**
   - `canvas-area` 不再允许原生滚动
   - 画板在任意缩放级别下都可以拖动
   - 移除“只有放大到一定程度后才能拖动”的限制

7. **样式预设管理器瘦身与布局优化**
   - 去掉 `style-preset-manager-desc`
   - 去掉 `manager-hint`
   - 在现有“展示列数”基础上增加“展示行数”设置
   - `gradient-card` 改为更稳定的 `flex` 布局，约束宽度范围与间距

## 本轮信息架构建议

为保证界面职责清晰，本轮调整后建议形成如下结构：

### 顶部区域
- 顶部主工具区保留原有文件级操作
- 画布上方新增项目标签栏
- 标签栏负责：新建标签、切换标签、关闭标签、显示当前活动项目

### 左侧区域
- 左侧 `panel` 保留高频插入能力
- 默认展开
- 支持一键收起，收起后尽可能将更多空间还给画布

### 中央区域
- 中央仍为主画布编辑区
- 取消 `canvas-area` 原生滚动
- 视口拖动能力在任意倍率下都可用

### 右侧区域
- 右侧固定面板聚焦核心编辑能力
- 建议收敛为：
  - 属性
  - 图层
  - 历史
- “预览 / 检查”不再作为右侧固定标签项常驻

### 画布右下角状态区
- 在 `Canvas / SVG` 切换器左侧放置：
  - 预览图标
  - 检查图标
- 在 `Canvas / SVG` 切换器上保留模式切换能力
- 让状态查看、问题检查、视图切换形成一个统一的轻量工具组

## Milestone 1：左侧工作台可伸缩化

**目标：让左侧高频工具既保留可达性，又能在需要时快速让位给画布。**

### 1.1 左侧 panel 收起 / 展开入口
- 在编辑工具栏底部新增切换图标
- 展开状态下显示“收起”图标
- 收起状态下显示“展开”图标
- 默认进入页面时处于展开状态

### 1.2 动画与状态反馈
- 收起 / 展开需要提供平滑过渡动画
- 建议动画覆盖：
  - 面板宽度变化
  - 内容透明度变化
  - 画布区域自适应伸缩
- 动画应避免突兀跳变或布局闪烁

### 1.3 布局影响范围
- 左侧面板收起后，主画布区域应扩展
- 不应影响右侧面板和顶部工具栏的稳定布局
- 收起 / 展开不应打断当前编辑状态

**预期产出：**
- 小屏或复杂项目下，画布可获得更大可视区域
- 用户能在“插入效率”和“编辑空间”之间快速切换

## Milestone 2：Canvas / SVG 视图与状态入口升级

**目标：把底部模式区从“单纯切换按钮”升级为“视图 + 状态 + 诊断”的统一入口。**

### 2.1 Canvas / SVG 模式区重构
- 保留 `Canvas` 视图入口
- `SVG` 入口升级为可提供 `tooltips` 的二级视图入口
- 建议底部工具组结构为：
  - 预览图标
  - 检查图标
  - `Canvas`
  - `SVG`

### 2.2 SVG 图形模式
- 以图形方式预览当前 SVG 输出结果
- 更适合用于快速比对导出效果与画布结果是否一致
- 重点关注视觉一致性，而不是编辑

### 2.3 SVG 代码模式
- 只读展示当前 SVG 代码
- 需要提供代码高亮
- 代码模式属于“预览型输出视图”，当前阶段不进入可编辑范围

### 2.4 预览 / 检查图标化
- 将右侧固定面板中的“预览 / 检查”迁出
- 改为底部图标入口，减少右侧面板占用
- 图标需要提供明确 `tooltip` 文案，降低学习成本

### 2.5 检查问题列表与空状态
- `hover` 检查图标时展示问题列表
- 无问题时展示空状态，而不是空白浮层
- 空状态需要让用户明确知道“当前没有检测到问题”

**预期产出：**
- 右侧面板职责更聚焦
- 模式切换与问题诊断更贴近画布使用场景
- `SVG` 输出查看路径更短、更明确

---

## Milestone 3：画布右键菜单与图层菜单统一

**目标：统一对象上下文操作入口，降低认知切换成本。**

### 3.1 画布对象右键行为补齐
- 当用户在画布上右键某个图形对象时
- 展示与图层面板中右键该对象一致的菜单项

### 3.2 菜单能力对齐
建议至少保持以下能力一致：
- 复制
- 删除
- 锁定 / 解锁
- 上移 / 下移
- 置顶 / 置底
- 编组 / 解组
- 保存到素材库

### 3.3 一致性要求
- 同一对象在画布区与图层区触发右键时，菜单语义保持一致
- 不同选择态（单选 / 多选 / 锁定态）下的菜单状态也应保持一致
- 避免后续出现两套菜单配置分叉维护

**预期产出：**
- 用户不需要记忆“在哪个区域右键才能看到哪个菜单”
- 对象操作心智更统一

---

## Milestone 4：多标签项目工作流

**目标：支持像专业设计工具一样并行处理多个项目。**

### 4.1 顶部标签栏
- 画布顶部新增标签页栏
- 每个标签页对应一个独立项目
- 当前活动项目以高亮标签显示

### 4.2 基础标签能力
V1 建议至少支持：
- 新开标签页
- 切换标签页
- 关闭标签页
- 保持每个标签独立的项目内容

### 4.3 状态隔离要求
每个标签至少需要独立维护：
- 画布内容
- 当前选中对象
- 缩放与视口位置
- 历史记录
- 当前视图模式（如 `Canvas / SVG`）

### 4.4 交互体验目标
- 切换标签时不丢失当前项目状态
- 新标签打开路径清晰，接近 Photoshop 的使用直觉
- 若存在未保存内容，建议在标签上提供轻量提示

**预期产出：**
- 支持多项目并行设计
- 显著降低频繁切项目时的操作成本

---

## Milestone 5：画布视口交互修复

**目标：让画布拖动与视口控制更加稳定、可预期。**

### 5.1 canvas-area 取消原生滚动
- `canvas-area` 不再出现原生滚动行为
- 用户对画布位置的控制统一回到视口平移逻辑中
- 避免滚动条与缩放 / 拖动逻辑互相干扰

### 5.2 移除拖动阈值限制
- 当前“只有放大到一定程度后才能拖动”的限制需要移除
- 无论当前画布是否超出容器，都允许拖动视口

### 5.3 与现有交互的兼容性
- 需要兼容已有空格拖动画布能力
- 需要避免与对象拖拽、节点编辑、框选等行为冲突
- 在不同缩放级别下保持拖动手感一致

**预期产出：**
- 画布始终可被主动定位
- 视口交互不再受容器滚动状态影响
- 编辑手感更接近专业画布工具

---

## Milestone 6：样式预设管理器瘦身与布局优化

**目标：移除冗余说明，提升管理界面密度与可配置性。**

### 6.1 文案瘦身
- 移除 `style-preset-manager-desc`
- 移除 `manager-hint`
- 管理界面尽量只保留标题、字段、列表本体和必要操作

### 6.2 展示配置增强
- 在现有“展示列数”基础上增加“展示行数”设置
- 行数和列数共同决定管理器中的可视布局密度
- 保持相关配置继续持久化

### 6.3 渐变卡片布局优化
- `gradient-card` 最小宽度：`330px`
- `gradient-card` 最大宽度：`450px`
- 使用 `flex` 布局
- 卡片间距固定为 `16px`
- 在不同容器宽度下尽量保持稳定、整齐的换行效果

**预期产出：**
- 管理器界面更简洁
- 视觉层次更集中在“内容本身”和“布局控制”上
- 渐变卡片在中大尺寸面板内显示更稳定

## 风险与实现注意事项

1. **多标签项目会显著影响状态管理边界**
   - 需要提前梳理每个标签的独立状态范围
   - 尤其要关注历史记录、选中态、导出数据与视图模式的隔离

2. **右键菜单统一建议复用同一份菜单定义**
   - 避免画布区和图层区各自维护一套菜单项
   - 否则后续扩展容易再次分叉

3. **视口拖动逻辑需要重新检查事件优先级**
   - 重点关注与对象拖拽、节点编辑、框选、滚轮缩放之间的冲突

4. **SVG 代码模式当前建议保持只读**
   - 本轮目标是“可预览、可检查、可高亮”
   - 暂不扩展到代码编辑与双向同步，避免范围失控
