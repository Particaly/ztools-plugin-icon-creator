# Icon Creator 功能路线图

## 项目现状总览

Icon Creator 是一个功能完善的图标编辑工具，基于 Vue 3 + FabricJS + PathKit 构建。

### 已实现核心能力

**编辑基础**
- ✅ 多画板支持与管理
- ✅ 22+ 种基础形状库
- ✅ 文字编辑（多预设）
- ✅ 图片导入
- ✅ 图层管理（排序、显示/隐藏、锁定）
- ✅ 编组/解组
- ✅ 撤销/重做系统

**高级编辑**
- ✅ 布尔运算（并集、交集、差集、异或）
- ✅ 钢笔工具
- ✅ 路径直接编辑（点位模式、线段模式）
- ✅ 对象变换（位置、尺寸、旋转）
- ✅ 对齐与等距分布
- ✅ 对齐参考线（拖动时自动吸附）

**样式系统**
- ✅ 填充与描边
- ✅ 纯色与渐变（线性、径向）
- ✅ 颜色预设管理
- ✅ 渐变预设管理
- ✅ 透明度控制

**资产与模板**
- ✅ 图标模板库（App Icon、Favicon、徽章等）
- ✅ 用户素材库
- ✅ Iconify 集成（搜索与导入）
- ✅ SVG 粘贴导入

**导出与交付**
- ✅ SVG 导出（可选背景）
- ✅ PNG 多尺寸导出
- ✅ 导出预设（Favicon、PWA、Android、iOS、Electron）
- ✅ 批量导出（所有画板打包为 ZIP）
- ✅ 复制为 SVG/PNG

**辅助工具**
- ✅ 标尺与像素网格
- ✅ 安全区参考线（Keyline）
- ✅ 图标规范检查（安全区溢出、细描边、非整数坐标、颜色数量）
- ✅ 多尺寸预览面板
- ✅ 快捷键系统（可自定义）
- ✅ 工程文件保存/加载

## 功能增强路线图

以下按优先级和实施难度规划，分为四个阶段。

---

## 🎯 Phase 1: 视觉效果增强（高优先级）

### 1.1 阴影与模糊效果
**问题**：当前无法为图形添加投影、内阴影或模糊效果，限制了图标的视觉表现力。

**目标**
- 支持投影（Drop Shadow）
  - X/Y 偏移
  - 模糊半径
  - 扩展（Spread）
  - 颜色与不透明度
- 支持内阴影（Inner Shadow）
- 支持模糊效果（Gaussian Blur）
- 每个对象支持多层效果叠加

**技术方案**
- FabricJS 原生支持 `shadow` 属性
- 使用 CSS `filter` 或 SVG `<filter>` 实现模糊
- 属性面板增加"效果"折叠面板

**验收标准**
- [ ] 可在属性面板添加/删除/编辑阴影
- [ ] 阴影在画布实时预览
- [ ] 导出 PNG/SVG 时正确保留效果
- [ ] 撤销/重做支持效果修改

---

### 1.2 蒙版功能
**问题**：无法实现非破坏性的形状裁切，例如用圆形裁切图片。

**目标**
- 支持剪切蒙版（Clipping Mask）
- 支持透明度蒙版（Alpha Mask）
- 蒙版对象与被蒙版对象的层级关系清晰

**实现要点**
- 右键菜单增加"创建剪切蒙版"
- 图层面板显示蒙版层级关系
- 蒙版对象可独立编辑

**验收标准**
- [ ] 选中两个对象可创建蒙版
- [ ] 蒙版效果在画布正确显示
- [ ] 图层面板显示蒙版关系
- [ ] 导出时正确应用蒙版
- [ ] 可释放蒙版恢复原始对象

---

### 1.3 高级变换
**问题**：缺少常用的图形变换能力，如翻转、倾斜。

**目标**
- 水平翻转
- 垂直翻转
- 倾斜变换（Skew X/Y）
- 一键重置变换

**技术方案**
- FabricJS 原生支持 `flipX`/`flipY`/`skewX`/`skewY`
- 在属性面板或右键菜单提供入口

**验收标准**
- [ ] 可通过快捷操作翻转对象
- [ ] 倾斜值可精确输入
- [ ] 变换后边界框正确显示
- [ ] 导出时变换正确应用

---

## 🚀 Phase 2: 编辑效率提升（中高优先级）

### 2.1 智能对齐与分布增强
**问题**：当前对齐功能较基础，缺少智能间距和参考对象选择。

**目标**
- 智能间距（Smart Spacing）
  - 自动建议等间距值
  - 显示间距数值标注
- 相对于选区对齐 vs 相对于画布对齐
- 对齐到画板中心的快捷操作
- 分布时可选"按对象边缘"或"按对象中心"

**验收标准**
- [ ] 间距标注在拖动时实时显示
- [ ] 对齐参考可切换（选区/画布）
- [ ] 分布选项在 UI 中可配置

---

### 2.2 样式复制与批量编辑
**问题**：修改多个对象的样式需要逐个操作，效率低。

**目标**
- 样式吸管（取色器）
  - 点击任意对象吸取填充色、描边色
- 样式复制/粘贴
  - 复制对象的所有样式属性
  - 粘贴到其他对象
- 批量修改
  - 选中多个对象时可同时修改颜色、描边等

**技术方案**
- 工具栏增加吸管工具
- 快捷键支持（Cmd/Ctrl+Shift+C 复制样式，Cmd/Ctrl+Shift+V 粘贴样式）

**验收标准**
- [ ] 吸管工具可取色
- [ ] 样式可跨对象复制粘贴
- [ ] 多选时属性面板显示混合值状态
- [ ] 批量修改立即生效

---

### 2.3 形状编辑增强
**问题**：基础形状不可调节，例如圆角矩形的圆角半径固定。

**目标**
- 圆角矩形：可调节圆角半径（支持四角独立调节）
- 星形：可调节角数、内外半径比
- 多边形：可调节边数
- 圆弧/扇形：可调节起始角度和结束角度
- 实时拖动控制点调节（类似 Figma）

**技术方案**
- 为特定形状增加控制点手柄
- 属性面板提供数值输入
- 参考 Figma 的形状编辑交互

**验收标准**
- [ ] 圆角矩形支持四角独立圆角
- [ ] 星形、多边形参数可调
- [ ] 控制点拖动实时预览
- [ ] 调节后仍可再次编辑

---

### 2.4 对象锁定增强
**问题**：锁定对象后完全不可选中，某些场景不便。

**目标**
- 锁定位置（可选中但不可移动）
- 锁定尺寸（可移动但不可缩放）
- 锁定宽高比（已有，但入口不明显）
- 完全锁定（当前行为）

**验收标准**
- [ ] 锁定选项在右键菜单或属性面板可见
- [ ] 各锁定模式行为正确
- [ ] 图层面板显示锁定状态图标

---

## 🎨 Phase 3: 资产管理优化（中优先级）

### 3.1 颜色管理增强
**问题**：颜色预设管理较简单，缺少项目级色板和色彩空间支持。

**目标**
- 项目色板（随工程文件保存）
- 全局色板（跨项目共享）
- 颜色变量系统
  - 定义颜色变量
  - 对象引用变量
  - 修改变量后所有引用对象自动更新
- 支持 HSL/HSB 色彩空间
- 渐变编辑器增强
  - 色标位置精确输入
  - 渐变角度可视化调节

**验收标准**
- [ ] 可定义和使用颜色变量
- [ ] 变量修改时引用对象同步更新
- [ ] 色彩空间切换正常
- [ ] 渐变编辑器支持精确控制

---

### 3.2 组件系统
**问题**：重复使用的图标元素需要复制粘贴，修改时需要逐个更新。

**目标**
- 组件定义（Component/Master）
  - 将对象或编组定义为组件
  - 保存到组件库
- 组件实例（Instance）
  - 从组件库插入实例
  - 实例与主组件保持链接
- 组件更新
  - 修改主组件后所有实例自动更新
  - 实例可覆盖部分属性（颜色、尺寸）
- 组件库管理
  - 列表展示
  - 搜索与分类
  - 删除与重命名

**技术方案**
- 组件数据结构设计
- 实例与主组件的链接关系维护
- 工程文件格式扩展以支持组件

**验收标准**
- [ ] 可创建组件并插入实例
- [ ] 主组件修改后实例同步
- [ ] 实例可覆盖特定属性
- [ ] 组件库 UI 清晰易用
- [ ] 组件随工程保存/加载

---

### 3.3 图层组织增强
**问题**：复杂项目中图层平铺，难以组织。

**目标**
- 图层搜索与过滤增强
  - 按类型过滤（形状/文字/图片）
  - 按颜色过滤
  - 按效果过滤（有阴影/有描边等）
- 图层分组命名与折叠
  - 编组支持命名
  - 多级编组折叠展开
- 图层缩略图可选显示/隐藏
- 图层标签或颜色标记

**验收标准**
- [ ] 过滤功能准确
- [ ] 图层树支持多级折叠
- [ ] 缩略图开关生效
- [ ] 标签或颜色标记可应用

---

### 3.4 资产导入增强
**问题**：仅支持图片和 SVG，缺少其他格式支持。

**目标**
- PDF 导入（首页转为矢量对象）
- AI/EPS 文件导入（如果可行）
- SVG Sprite 导入（一次导入多个图标）
- 拖拽文件夹批量导入

**验收标准**
- [ ] PDF 首页正确转换
- [ ] 批量导入流程顺畅
- [ ] 导入对象可编辑

---

## 📊 Phase 4: 性能与体验优化（持续进行）

### 5.1 性能优化
**已知问题**
- 对象数量多时（>100）画布操作卡顿
- 大尺寸图片导入后编辑性能下降
- 撤销历史过长时内存占用大

**优化方向**
- 画布对象虚拟化（仅渲染可见区域）
- 大图片自动压缩或降采样
- 撤销历史限制与清理策略
- Web Worker 支持（布尔运算等计算密集任务）

---

### 5.2 用户体验优化
**细节改进**
- 首次启动引导教程
- 空状态优化（无画板、无图层时的提示）
- 操作提示与快捷键提示
- 错误提示更友好
- 加载动画与骨架屏
- 深色模式支持

---

### 5.3 可访问性
**目标**
- 键盘导航支持
- 屏幕阅读器适配
- 高对比度模式
- 快捷键冲突检测与提示

## 📝 实施注意事项

### 技术风险
1. **阴影/模糊导出兼容性**：确保 SVG/PNG 导出时效果一致
2. **组件系统复杂度**：需要设计良好的数据结构和同步机制
3. **性能优化**：对象多时需要虚拟化等技术，实现复杂

### 用户体验考虑
1. **学习曲线**：新功能需要引导和文档支持
2. **向后兼容**：新版本需要兼容旧工程文件
3. **功能可见性**：复杂功能需要渐进式呈现，避免界面臃肿

### 开发资源
- 每个 Phase 预计 2-4 周开发时间
- 建议按优先级逐步实施，每完成一个功能进行充分测试
- 关键功能可先做 MVP 验证，再完善细节

---

## 🎉 总结

Icon Creator 已经是一个功能完善的图标编辑工具，本路线图重点关注：

1. **视觉表现力**：阴影、模糊、蒙版等效果
2. **编辑效率**：样式复制、形状编辑、批量操作
3. **资产复用**：组件系统、颜色变量
4. **性能与体验**：性能优化、用户体验改进

建议优先实现 Phase 1 和 Phase 2 的功能，这些能显著提升工具的实用性和竞争力。Phase 3 的组件系统是中长期目标，需要更多设计和开发投入。Phase 4 和 Phase 5 持续优化性能、体验和代码质量。
